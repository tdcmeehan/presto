  <action>
Create integration test class that:
1. Extends IcebergDistributedSmokeTestBase (reuses Iceberg query runner setup)
2. Creates partitioned test tables in @BeforeClass:
   - `fact_orders` table partitioned by `order_date` with 10+ partitions
   - `dim_customers` table with selective filter values
   - Insert test data where ~70% of partitions can be pruned by a typical join filter
3. Implements test methods:
   - `testDynamicPartitionPruningMetrics()`: Execute join query with DPP enabled, assert DYNAMIC_FILTER_SPLITS_EXAMINED > 0 and DYNAMIC_FILTER_SPLITS_PRUNED > 0
   - `testDynamicPartitionPruningResultCorrectness()`: Execute same query with DPP enabled vs disabled, assert identical results
   - `testDynamicPartitionPruningDisabled()`: Execute query with DPP disabled, assert DYNAMIC_FILTER_SPLITS_PRUNED = 0
4. Use helper method `executeWithDppSession(boolean enabled, String sql)` that:
   - Creates session with `dynamic_partition_pruning_enabled` set appropriately
   - Uses `dynamic_partition_pruning_max_wait_time = '5s'` when enabled
   - Calls `getDistributedQueryRunner().executeWithQueryId()` and returns `ResultWithQueryId`

Use pattern from IcebergDistributedSmokeTestBase.testRuntimeMetricsReporter() for metrics retrieval:
```java
RuntimeStats runtimeStats = distributedQueryRunner.getCoordinator()
        .getQueryManager()
        .getFullQueryInfo(result.getQueryId())
        .getQueryStats()
        .getRuntimeStats();
```

The test query should be a star-schema join like:
```sql
SELECT f.order_id, f.amount, c.customer_name
FROM fact_orders f
JOIN dim_customers c ON f.customer_id = c.customer_id
WHERE c.region = 'WEST'
```
Where dim_customers with region='WEST' represents ~30% of customers, enabling ~70% partition pruning.

IMPORTANT: Import session property constants from SystemSessionProperties:
- DYNAMIC_PARTITION_PRUNING_ENABLED
- DYNAMIC_PARTITION_PRUNING_MAX_WAIT_TIME
