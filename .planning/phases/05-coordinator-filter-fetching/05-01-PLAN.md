- Guava ListenableFuture for async

**Established patterns:**
- TaskInfoFetcherWithEventLoop pattern for HTTP fetching
- RequestErrorTracker for backoff/retry
- StateMachine for state change listeners
- SimpleHttpResponseCallback for async HTTP

**Constraining decisions:**
- Phase 4: GET /v1/task/{taskId}/outputs/filter/{version} endpoint returns Map<String, TupleDomain<String>>
- Phase 4: TaskStatus.outputsVersion tracks filter version changes

**Note on column handle resolution:**
The worker stores TupleDomain<String> (column names), but CoordinatorDynamicFilter.addPartition() expects TupleDomain<ColumnHandle>. This conversion is deferred to Phase 6 (Scheduler Wiring) where the filter-to-connector mapping occurs. For this plan, we'll store the raw TupleDomain<String> and let Phase 6 handle the conversion.
