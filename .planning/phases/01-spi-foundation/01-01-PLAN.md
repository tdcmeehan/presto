  <files>presto-spi/src/main/java/com/facebook/presto/spi/connector/DynamicFilter.java</files>
  <action>
Create a new `DynamicFilter` interface in the connector package with these elements:

1. **Static constants:**
   - `EMPTY` - a DynamicFilter that is already complete with `TupleDomain.all()` (no filtering)

2. **Methods:**
   - `Set<ColumnHandle> getColumnsCovered()` - returns columns this filter applies to
   - `ListenableFuture<TupleDomain<ColumnHandle>> getConstraint()` - returns future that completes when filter is ready
   - `Duration getWaitTimeout()` - recommended wait duration from scheduler
   - `boolean isComplete()` - returns true if the filter future is already done
   - `boolean isBlocked()` - returns !isComplete() (convenience method)

3. **Static factory method:**
   - `static DynamicFilter empty()` - returns EMPTY constant

**Implementation Notes:**
- Use `com.google.common.util.concurrent.ListenableFuture` (already in SPI dependencies)
- Use `io.airlift.units.Duration` (already in SPI dependencies)
- The EMPTY constant should return:
  - Empty set for columns covered
  - Already-completed future with `TupleDomain.all()` for constraint
  - `Duration.ZERO` for wait timeout
  - `true` for isComplete()

**Style:**
- Follow Presto code style (see CLAUDE.md)
- Use static imports for `requireNonNull`
- Add Javadoc for public interface methods

Add a new default method to ConnectorSplitManager that accepts a DynamicFilter parameter:

```java
/**
 * Returns a split source for the given table layout with dynamic filter support.
 *
 * <p>The dynamic filter provides runtime filter constraints collected from join build sides.
 * Connectors can use this to prune partitions and files during split generation.
 *
 * <p>The default implementation ignores the dynamic filter and delegates to the
 * existing getSplits method, allowing existing connectors to work unchanged.
 *
 * @param transactionHandle the transaction handle
 * @param session the connector session
 * @param layout the table layout handle
 * @param splitSchedulingContext scheduling context
 * @param dynamicFilter runtime filter from join build side, or DynamicFilter.EMPTY if none
 * @return split source for the table
 */
default ConnectorSplitSource getSplits(
        ConnectorTransactionHandle transactionHandle,
        ConnectorSession session,
        ConnectorTableLayoutHandle layout,
        SplitSchedulingContext splitSchedulingContext,
        DynamicFilter dynamicFilter)
{
    return getSplits(transactionHandle, session, layout, splitSchedulingContext);
}
```

**Implementation Notes:**
- Default implementation ignores dynamicFilter and calls existing 4-arg getSplits
- This ensures all existing connectors continue to work without modification
- Connectors that want to use dynamic filters will override this method
- Place this method after the existing getSplits method (around line 31)

**Do NOT:**
- Modify the existing getSplits method signature
- Add any imports that aren't needed (DynamicFilter will be in same package)
