  <files>presto-main-base/src/main/java/com/facebook/presto/execution/scheduler/CoordinatorDynamicFilter.java</files>
  <action>
Create CoordinatorDynamicFilter implementing DynamicFilter interface. Model after LocalDynamicFilter but:

1. Use CompletableFuture<TupleDomain<ColumnHandle>> instead of SettableFuture (matches SPI)
2. Key by ColumnHandle (from connector) not VariableReferenceExpression (planner internal)
3. Thread-safe accumulation with synchronized addPartition()
4. Store columns covered, wait timeout, and expected partition count

Constructor parameters:
- Set<ColumnHandle> columnsCovered - columns this filter applies to
- Duration waitTimeout - from session property
- int expectedPartitions - number of workers/partitions expected

Methods:
- getColumnsCovered() - return stored columns
- getConstraint() - return the CompletableFuture
- getWaitTimeout() - return stored timeout
- isComplete() - check if future is done
- addPartition(TupleDomain<ColumnHandle>) - synchronized, accumulate and merge when all received

Use TupleDomain.columnWiseUnion() for merging, same as LocalDynamicFilter.
Do NOT convert between ColumnHandle and VariableReferenceExpression - keep it connector-level.
