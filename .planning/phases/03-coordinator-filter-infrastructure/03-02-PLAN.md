  <files>presto-main-base/src/main/java/com/facebook/presto/execution/scheduler/DynamicFilterService.java</files>
  <action>
Create DynamicFilterService as a simple registry for CoordinatorDynamicFilter instances.

Structure:
- @ThreadSafe annotation
- Store filters in nested map: ConcurrentMap<QueryId, ConcurrentMap<String, CoordinatorDynamicFilter>>
  - Outer key: QueryId (query scope)
  - Inner key: filterId (String, from JoinNode.getDynamicFilters())

Methods:
- registerFilter(QueryId, String filterId, CoordinatorDynamicFilter) - register a filter
- getFilter(QueryId, String filterId) -> Optional<CoordinatorDynamicFilter> - lookup by query+filter ID
- removeFiltersForQuery(QueryId) - cleanup all filters for a completed query

Keep it simple - no background threads, no expiration logic. Query cleanup is caller's responsibility (will be called from QueryStateMachine state change listener in later phase).

Do NOT add Guice @Inject yet - that comes when we wire into the scheduler.
